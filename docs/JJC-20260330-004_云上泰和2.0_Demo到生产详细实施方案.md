- 期货/现货/采购数据均来自模拟

### 2.2 Agent运行时选型

**结论：继续使用 OpenClaw，不换框架**

理由：
1. 已有完整的三省六部Agent配置，迁移成本高
2. OpenClaw已支持subagent调度、飞书channel、skill机制
3. 其skill机制完全支持自定义tool（Python/JS实现）
4. MiniMax作为主要LLM provider已集成

**核心工作不是换框架，而是给OpenClaw补完tool执行能力**

### 2.3 多Agent协作机制

沿用现有三省六部流程，扩展执行层：

```
皇上旨意 → 太子(分拣) 
         → 中书省(起草方案+分配技能)
         → 门下省(审核)
         → 尚书省(派发任务+绑定Skill工具)
         → 六部Agent(执行业务Skill + 调用Tool)
         → 数据层(业务API/数据库)
         → 回奏
```

Skill执行时Agent间通过 `sessions_send` 传递上下文，无需Event Bus重构。

## 三、技能实现路径（核心工作）

### 3.1 SKILL.md → 可执行代码的桥接方案

**当前模式（无效）**：
```
SKILL.md（只有描述） → LLM自己猜怎么执行 → 结果不可控
```

**目标模式（可控）**：
```
SKILL.md（描述+工具声明）
    ↓
Skill实现文件（Python代码，具体tool函数）
    ↓
OpenClaw Tool Registry（注册到runtime）
    ↓
Agent调用时真正执行tool代码
```

### 3.2 Skill实现模板

每个业务Skill需要两个文件：

**文件1：`skills/xxx/SKILL.md`**（已存在，补充tools字段）
```markdown
---
name: futures-analysis
description: 期货行情技术分析与趋势研判
tools:
  - fetch_futures_quotes      # 获取期货实时行情
  - calculate_technical_indicators  # 计算技术指标
  - generate_trading_signal   # 产生交易信号
---
```

**文件2：`skills/xxx/tools.py`**（新增，可执行代码）
```python
# skills/futures-spot/tools.py

    return resp.json()

def calculate_technical_indicators(symbol: str, period: int = 20) -> Dict[str, float]:
    """计算技术指标（MA, RSI, MACD）"""
    # 实现逻辑...
    pass

TOOL_REGISTRY = {
    "fetch_futures_quotes": fetch_futures_quotes,
    "calculate_technical_indicators": calculate_technical_indicators,
}
```

### 3.3 工具注册到OpenClaw

OpenClaw通过 `tools` 目录自动发现tool：
```
~/.openclaw/
├── workspaces/
│   ├── workspace-taizi/         # 太子
│   ├── workspace-zhongshu/       # 中书省
│   └── workspace-shangshu/       # 尚书省
├── tools/                        # 全局共享工具
│   ├── feishu_tools.py
│   ├── dingtalk_tools.py
│   ├── business_data_tools.py
│   └── skill_executor.py
└── skills/                       # 各Agent的skill实现
    ├── futures-spot/
    │   ├── SKILL.md
    │   └── tools.py
    └── ...
```

**tool自动发现机制**：
OpenClaw读取skill目录下的 `tools.py`，从中提取函数注册为该skill的可用tool。

### 3.4 多Agent协同工作流程

**示例：期现交易部执行"期货分析"任务**

```
1. 尚书省派发任务给期现交易部Agent
   sessions_send(to="期现交易部", task="分析CU2506期货走势")

2. 期现交易部Agent加载futures-spot skill
   → 读取SKILL.md，发现需要tools: fetch_futures_quotes
   
3. Agent调用tool执行
   → 执行 fetch_futures_quotes(symbol="CU2506")
   → 返回行情数据
   
4. Agent分析数据，调用 calculate_technical_indicators
   → 返回MA、RSI、MACD

5. Agent生成分析报告，调用 generate_trading_signal
   → 返回信号：买入/卖出/持有

6. 回奏尚书省
```

## 四、接入层设计

### 4.1 钉钉接入方案

**架构**：
```
钉钉企业 → 钉钉机器人 → OpenClaw DingTalk Channel → Gateway → Agent
```

**实施步骤**：

**Step 1：创建钉钉机器人**
1. 登录钉钉开放平台：open.dingtalk.com
2. 创建企业内部应用 → 添加机器人
3. 配置消息接收模式：**Stream模式**（推荐，无需公网回调）
4. 获取：`AppKey`、`AppSecret`、`BotCode`

**Step 2：开发DingTalk Channel Plugin for OpenClaw**
```
~/.openclaw/extensions/
└── dingtalk/
    ├── dingtalk_channel.py    # 主channel实现
    ├── dingtalk_client.py     # 钉钉API封装
    └── skill.yaml             # channel配置
```

**Step 3：配置OpenClaw使用钉钉**
```yaml
# ~/.openclaw/config.yml
channels:
  dingtalk:
    enabled: true
    app_key: "dingXXXXX"
    app_secret: "XXXXX"
    bot_code: "taihe2_bot"
```

**Step 4：注册到Gateway并测试**
```bash
openclaw channel add dingtalk
openclaw gateway restart
```

### 4.3 权限系统设计

**三级权限**：

| 级别 | 角色 | 能力 |
|------|------|------|
| L1 | 普通员工 | 仅能对话、查询（read-only） |
| L2 | 部门主管 | 对话 + 发起业务操作（如提交采购申请） |
| L3 | 管理员 | 所有能力 + 配置管理 |

**实现方式**：
- OpenClaw workspace内建权限（已支持）
- 飞书/钉钉企业权限映射（部门负责人 = L2）
- API接口加token验证（简单JWT即可）

### 5.2 数据隔离方案

**按部门数据隔离**：
- 每个部门Agent只配置本部门的数据源凭证
- 数据库账号按部门授权（MySQL/PG行级隔离）
- API Key按业务线分开

**实现**：
```python

def fetch_procurement_data(dept_id: str, date_range: tuple):
    # 校验dept_id属于当前Agent权限范围
    if dept_id not in current_agent.allowed_depts:
        raise PermissionError("无权访问该部门数据")
    # 查询...
```

### 5.3 API接口设计

**统一业务API网关**：

```
Agent Tool调用
    ↓
/api/v1/business/{domain}/{action}
    ↓
业务系统（ERP/期货API/物流API）
    ↓
返回结构化数据
```

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/futures/quotes` | GET | 获取期货行情 |
| `/api/v1/futures/indicators` | POST | 计算技术指标 |
| `/api/v1/procurement/orders` | GET | 采购订单查询 |
| `/api/v1/finance/reports` | GET | 财务报表 |
| `/api/v1/supply/logistics` | GET | 物流跟踪 |
| `/api/v1/fx/rates` | GET | 实时汇率 |

**安全**：
- 所有API通过HTTPS访问
- 敏感凭证存环境变量，不硬编码
- API请求经Agent tool层做参数校验

## 八、关键风险与对策

| 风险 | 等级 | 对策 |
|------|------|------|
| 业务API不稳定/无文档 | 高 | 先用模拟数据接口过渡，同步协调API提供方 |
| 期货数据API需要付费 | 中 | 评估免费额度，或申请公司采购 |
| 钉钉Stream模式需要企业认证 | 中 | 确认公司钉钉已认证，否则用 Callback模式 |
| Agent执行tool产生财务损失 | 极高 | L2级以上操作必须人工确认，禁止自动执行交易 |
| LLM幻觉导致错误业务判断 | 高 | 关键数据必须tool返回，禁止纯LLM推断 |
