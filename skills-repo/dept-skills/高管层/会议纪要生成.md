# 会议纪要生成

## 描述
接收会议音频或文字记录，自动生成结构化会议纪要，并提取待办事项分派给相关部门。

## 触发条件
- 用户发送会议记录并要求"生成纪要"
- 用户说"帮我整理一下会议内容"
- 高管层会议结束后需要快速输出纪要

## 执行步骤
1. 接收会议文本内容（支持文字输入或音频转写）
2. 识别会议类型（战略决策会议/经营分析会议/业务协调会议）
3. 提取关键议题、讨论要点、决策结论
4. 识别并提取待办事项（Action Items），标注负责人和截止时间
5. 生成标准格式纪要文档
6. 调取尚书省流程，将待办事项分发至对应部门Agent

## 输出格式
```
# 会议纪要
- 会议名称：[name]
- 时间：[datetime]
- 参会人员：[list]
- 议题：[topic list]
## 讨论要点
1. [point 1]
2. [point 2]
## 决策结论
- [decision 1]
- [decision 2]
## 待办事项
| 序号 | 事项 | 负责人 | 截止时间 | 承接部门 |
|------|------|--------|----------|----------|
| 1    | ...  | ...    | ...      | ...      |
```

## 示例
输入：高管战略会议文字记录
输出：完整的会议纪要，包含"立即采购50万桶原油现货"的决策，以及"财务经营部在3天内完成资金调配方案"的待办

## 依赖
- 企业微信待办创建（wecom-edit-todo）
